# Qorix Sprint Capacity Calculator - Quick Start

> Get the calculator running in a few minutes.

## 1. Open the App

1. Open the `Sprint Capacity Planning` folder.
2. Double-click `index.html`.
3. The calculator opens in your browser.

If you are deploying to GitHub Pages, open the Pages URL instead of the local file.

## 2. Fill the Sprint Details

Go to **Configuration** and enter the required fields:

- Program / Project Name
- Team / Module Name
- Sprint Number
- Sprint Name
- Start Date
- End Date
- Working Days in Sprint
- Hours per Day

Optional fields:

- Scrum Master
- Product Owner

Click **Save Configuration**.

## 3. Add Team Members

Go to **Team Capacity** and click **+ Add Team Member**.

For each person, enter:

- Name
- Role
- Allocation %
- Productivity Factor %

Productivity Factor is capped at 95.

## 4. Plan Availability

Use the date-wise planner to mark each member for each day as:

- Full Day
- Half Leave
- Leave
- Holiday

Then click **Save Team Planning**.

## 5. Review the Dashboard

Open **Dashboard** to review:

- Total Team Members
- Total Capacity in Days
- Total Capacity in Hours
- Avg Daily Capacity
- Productivity Factor

## 6. Export or Print

- Click **Export CSV** to download the plan.
- Open **Export PDF** to print or save a PDF report.
- Use **Import CSV** to load a saved CSV file back into the app.

## 7. Reset When Needed

Use the top-right **Reset** button if you want to clear the saved configuration and team data.

## GitHub Deployment Checklist

- Make sure `index.html` is present in the repository.
- Commit `README.md`, `QUICK-START.md`, and `USER-GUIDE.md` with the app.
- Enable GitHub Pages from the repository settings.
- Use the published Pages URL for testing.
